Detailed Description

Class representing an EJDB query.

Valid queries can only be created via ejdb::db::create_query.

Should the parent ejdb::db object expire before the query, all operations performed on or with the query will fail, with any std::error_codes set to std::errc::bad_address (EFAULT). The parent ejdb::db object is guaranteed to stay alive for the duration of an operation.

query && ejdb::query::set_hints ( const jbson::document &  obj)

Sets hints for a query.

EJDB's hints documentation follows.

Available hints:

  • $max Maximum number in the result set
  • $skip Number of skipped results in the result set
  • $orderby Sorting order of query fields.
  • $fields Set subset of fetched fields If a field presented in $orderby clause it will be forced to include in resulting records. Example:
    {
    "$orderby" : { //ORDER BY field1 ASC, field2 DESC
    "field1" : 1,
    "field2" : -1
    },
    "$fields" : { //SELECT ONLY {_id, field1, field2}
    "field1" : 1,
    "field2" : 1
    }
    }
